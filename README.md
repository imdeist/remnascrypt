## 💡 О проекте

**Remnascrypt** перерос из простого установочного скрипта в полноценный интерактивный CLI-менеджер. Теперь вам не нужно запоминать флаги и команды. Скрипт сам настроит окружение, оптимизирует ядро сервера для проксирования, выпустит сертификаты и предоставит красивую панель управления нодой прямо в вашем терминале.

### ✨ Главные возможности

* 🎛️ **Интерактивный TUI-интерфейс:** Красивое визуальное меню для управления всеми аспектами ноды.
* 🛡️ **SelfSNI & SSL из коробки:** Автоматическая настройка Nginx и прозрачный выпуск Let's Encrypt сертификатов (`HTTP-01`).
* ⚡ **Xray Core Manager:** Установка, обновление и переключение версий ядра Xray (Stable/Beta) в один клик.
* 🚀 **Boost Network:** Автоматическое включение алгоритма TCP BBR и расширение буферов для максимальной скорости и минимального пинга.
* 🎭 **Fake-Site Integration:** Динамическая загрузка маскировочных лендингов-заглушек из [Capsite](https://github.com/imdeist/capsite).
* 🔧 **Live Dashboard:** Мониторинг версий (умный поиск в `package.json`), портов и статуса работы Docker/Nginx в реальном времени.
* 🧹 **Absolute Uninstaller:** Полное выжигание ноды, контейнеров и логов при необходимости очистить сервер.

## ⚙️ Требования перед установкой

1. **ОС:** Чистый сервер на Debian 11/12/13 или Ubuntu 20.04/22.04/24.04.
2. **Права:** Запуск строго от пользователя `root`.
3. **DNS:** Ваш домен **уже должен быть направлен** на IP-адрес сервера (A-запись).
4. **Порты:** Порты `80` и `443` должны быть свободны и открыты в Firewall (нужно для получения SSL).
5. **Remnawave Panel:** Вы должны заранее создать ноду в вашей панели управления и получить `SECRET_KEY`.

> **Где взять SECRET_KEY?**  
> В панели Remnawave перейдите в `Nodes` → `Management` → Нажмите `+`. Укажите домен и желаемый порт. Скопируйте сгенерированный ключ.

## 🚀 Быстрый запуск

Подключитесь к серверу по SSH от имени `root` и выполните команду:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/imdeist/remnascrypt/main/remnascrypt.sh)

```

Скрипт запустит мастер первичной установки, который шаг за шагом запросит ваш домен, порты и секретный ключ, после чего сделает всю грязную работу за вас.

## 🛠 Под капотом: Как это работает

При первичной установке `Remnascrypt` выполняет следующую последовательность действий:

1. **Check & Install:** Проверяет наличие `curl`, `certbot`, `docker` и других зависимостей. Если чего-то нет — устанавливает тихо и без мусора в консоли.
2. **Network Optimize:** Прописывает в `sysctl.d` оптимальные значения TCP-буферов (64MB), включает `fq` и алгоритм `bbr`.
3. **SSL Routing:** Поднимает временный Nginx на 80 порту для прохождения челленджа Certbot, получает сертификаты и разворачивает финальный конфиг:
* `80 порт` — жесткий редирект на HTTPS.
* `443 порт` — отдает фейк-сайт (маскировку).
* `SelfSNI порт` — проксирует защищенный трафик напрямую в контейнер ноды.


4. **Docker Compose:** Генерирует изолированный манифест с пробросом сертификатов в режиме Read-Only.
5. **System Link:** Создает глобальный алиас `remnascrypt` в `/usr/local/bin/`.

## 🧩 Ручное управление (Для продвинутых)

Если вам необходимо посмотреть логи руками или убедиться, что всё работает вне скрипта:

**Проверка работы контейнера:**

```bash
docker ps -a | grep remnascrypt

```

**Чтение логов ноды в реальном времени:**

```bash
docker compose -f /opt/remnascrypt/docker-compose.yml logs -f remnascrypt

```

## ⚠️ Возможные проблемы и решения

* **Ошибка сертификата при установке:** Убедитесь, что A-запись домена обновилась (обычно занимает от 2 до 15 минут) и что провайдер не блокирует 80-й порт.
* **Нода не появляется в панели (Offline):** Перепроверьте `SECRET_KEY` и `NODE_PORT` (пункты 5 и 6 в меню `remnascrypt`). Порт в панели и порт на сервере должны совпадать.
